# Claude Code — контекст проекта

## Что это

Веб-калькулятор Retention & LTV. Проект для личного бренда: выкатывается на GitHub Pages, описывается в посте, разбирается широкой аудиторией продуктовиков/маркетологов/менеджеров.

**Принципиально не black box.** Раздел методологии — отдельная полноценная страница с формулой power law, ограничениями модели и источниками каждого пресета.

## С чего начать

1. **Прочитай `docs/spec.md`** — там полная спецификация. Особенное внимание секциям:
   - §0 — карта документов
   - §3.1 Блок 4 — структура пресетов с quality × geo (35 вариантов)
   - §3.5 — страница методологии
   - §5 — стек и обоснование
   - §8 — структура проекта
2. **Прочитай `docs/methodology.md`** — это и source-of-truth по источникам, и контент страницы `/methodology` в приложении
3. **Прочитай `docs/presets.json`** — данные пресетов, которые будет грузить приложение

## Карта файлов

```
docs/spec.md          ← спецификация (что строим)
docs/methodology.md   ← методология + источники (рендерится в app)
docs/presets.json     ← данные пресетов (грузится в runtime)
README.md             ← витрина для GitHub
CLAUDE.md             ← этот файл
```

## Жёсткие ограничения

- **Хостинг:** только GitHub Pages (статика). Никакого бэкенда, БД, env-переменных
- **Не black box:** методология как полноценная страница с anchor-навигацией, не expandable-блок
- **Источники цифр:** живут только в `docs/methodology.md`. В `presets.json` поле `sources` намеренно отсутствует — не плодим дубли
- **ARPU нормализация:** канонический источник в пресете — `arpu_per_day`. Поля `arpu_monthly` / `arpu_annual` / `arpdau` — display-only

## Рекомендуемый стек (см. §5 ТЗ)

React 18 + Vite + Recharts + Tailwind + react-router-dom (HashRouter) + react-markdown.

Если в процессе видишь, что что-то лучше делать иначе — обсуди с пользователем перед тем как менять.

## Что делать в первую очередь

1. Инициализация проекта: `npm create vite@latest . -- --template react`
2. Подключи зависимости из §5 ТЗ
3. Настрой структуру `src/` как описано в §8 ТЗ
4. Скопируй `docs/presets.json` и `docs/methodology.md` в `public/` (или сделай build-step, который их туда кладёт). Vite сериализует в бандл только то, что в `public/` или импортируется из `src/`
5. Реализуй math-ядро (`src/lib/powerLaw.js`, `src/lib/ltv.js`) с юнит-тестами **до** UI — это самое критичное место с точки зрения корректности
6. Дальше — UI по §3 и §4 ТЗ

## Чего не делать без обсуждения

- Не добавлять backend / API
- Не уходить от GitHub Pages-совместимого билда
- Не править `methodology.md` без аккуратной причины — это публичный контент, видимый и в репо, и в приложении
- Не добавлять `sources` обратно в `presets.json` — источники есть в `methodology.md`
- Не делать SaaS B2B / Subscription Apps пресеты в дневной шкале (см. §10 ТЗ)
