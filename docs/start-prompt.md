Задачи на эту сессию (по порядку, пошагово):

1. Прочитай `CLAUDE.md`, потом `docs/spec.md` (полностью), потом `docs/methodology.md`, потом `docs/presets.json`. После каждого файла подтверди что прочитал. Не суммируй — просто скажи "прочитал, понял".

3. После ответов — предложи план реализации MVP первой итерации. MVP = Must have из §6 спецификации:
   - Core calculator (manual input)
   - KPI cards
   - Industry presets с quality × geo селекторами
   - Breakeven visualization
   - Methodology page (роут + рендер `methodology.md`)
   - Cohort P&L block

   План должен включать: порядок задач, какие файлы создаются, точки где нужно моё подтверждение перед движением дальше. Should have / Nice to have фичи — НЕ в MVP, оставляем на следующие итерации.

4. После моего "ок" на план — начинай с инициализации проекта (`npm create vite`, зависимости, структура `src/` из §8 спеки). Math-ядро (`powerLaw.js` + `ltv.js`) с юнит-тестами реализуй ДО UI — так заложено в `CLAUDE.md`.

Жёсткие ограничения (повторяю из `CLAUDE.md`):
- Только GitHub Pages, никакого бэкенда
- Methodology — отдельная страница с роутом, не expandable
- `presets.json` НЕ должен содержать поле `sources` (источники только в `methodology.md`)
- ARPU нормализация: канонический источник `arpu_per_day`, остальные — display-only

Если в процессе видишь что какая-то часть спеки устарела или противоречит другим документам — останавливайся и обсуждай со мной, не правь молча.
